# NC财务术语库配置模板

> 请根据此模板整理NC财务术语，用于导入SQLBot术语库。

---

## 使用说明

1. 按照下表格式整理术语
2. 导出为Excel文件（.xlsx）
3. 在SQLBot的「系统管理 → 术语配置」中导入
4. 术语描述要包含：业务定义、对应表/字段、SQL条件

---

## 一、会计科目类术语（10个）

| 术语名称 | 同义词（逗号分隔） | 术语描述 | 数据源名称 |
|:---|:---|:---|:---|
| 资产类科目 | 资产科目 | 科目编码以1开头的科目，包括现金、银行存款、应收账款、存货、固定资产等。判断条件：bd_accsubj.accsubjcode LIKE '1%'，余额方向：借方（balanorient = 0） | NC财务系统 |
| 负债类科目 | 负债科目 | 科目编码以2开头的科目，包括短期借款、应付账款、预收账款等。判断条件：bd_accsubj.accsubjcode LIKE '2%'，余额方向：贷方（balanorient = 1） | NC财务系统 |
| 权益类科目 | 所有者权益科目,净资产科目 | 科目编码以3开头的科目，包括实收资本、资本公积、盈余公积、未分配利润等。判断条件：bd_accsubj.accsubjcode LIKE '3%'，余额方向：贷方 | NC财务系统 |
| 成本类科目 | 成本科目 | 科目编码以4开头的科目，包括生产成本、制造费用等。判断条件：bd_accsubj.accsubjcode LIKE '4%' | NC财务系统 |
| 损益类科目 | 损益科目 | 科目编码以5或6开头的科目。收入类以5开头（主营业务收入等），费用类以6开头（管理费用、销售费用等）。判断条件：bd_accsubj.accsubjcode LIKE '5%' OR '6%' | NC财务系统 |
| 现金 | 库存现金,现金余额 | 库存现金科目的期末余额。科目编码：1001，数据来源：gl_balance.endbalance WHERE accsubjcode = '1001' | NC财务系统 |
| 银行存款 | 银行余额,银行账户余额 | 银行存款科目的期末余额。科目编码：1002，数据来源：gl_balance.endbalance WHERE accsubjcode LIKE '1002%' | NC财务系统 |
| 应收账款 | 应收余额,客户欠款 | 应收账款科目的期末余额，表示客户尚未支付的款项。科目编码：1122，数据来源：gl_balance.endbalance WHERE accsubjcode = '1122'，或通过ar_recbill收款单统计未核销金额 | NC财务系统 |
| 应付账款 | 应付余额,欠供应商款项 | 应付账款科目的期末余额，表示企业尚未支付给供应商的款项。科目编码：2202，数据来源：gl_balance.endbalance WHERE accsubjcode = '2202' | NC财务系统 |
| 末级科目 | 明细科目,可做账科目 | 只有末级科目（isleaf='Y'）才能进行账务处理。非末级科目是汇总值，不能直接记账。查询条件：bd_accsubj.isleaf = 'Y' | NC财务系统 |

| 期末余额 | 余额,当前余额 | 会计期间末的科目余额。数据来源：gl_balance.endbalance | NC财务系统 |

| 摘要 | 凭证摘要,分录摘要 | 凭证分录的业务说明。字段：gl_detail.explanation | NC财务系统 |
| 凭证号 | 凭证编号 | 凭证的顺序编号，格式通常为"类别-序号"。字段：gl_voucher.num | NC财务系统 |

## 三、期间与时间类术语（8个）

| 术语名称 | 同义词（逗号分隔） | 术语描述 | 数据源名称 |
|:---|:---|:---|:---|
| 本月 | 当月,当期 | 当前会计期间，由cyear和cmonth共同确定。条件：cyear = EXTRACT(YEAR FROM CURRENT_DATE) AND cmonth = EXTRACT(MONTH FROM CURRENT_DATE) | NC财务系统 |
| 上月 | 上期,前月 | 当前期间的前一个会计期间。注意跨年处理：如当期是1月，上期是上一年的12月 | NC财务系统 |
| 本季度 | 当季 | 当前所在季度。Q1: cmonth IN (1,2,3)，Q2: cmonth IN (4,5,6)，Q3: cmonth IN (7,8,9)，Q4: cmonth IN (10,11,12) | NC财务系统 |
| 本年 | 当年,年度 | 当前会计年度内的所有期间。条件：cyear = 当前年 AND cmonth BETWEEN 1 AND 12 | NC财务系统 |
| 去年同期 | 同比基期,上年同月 | 去年同一会计期间。条件：cyear = 当前年-1 AND cmonth = 当前月 | NC财务系统 |
| 同比 | 同期比较,与去年同期比 | 与去年同一期间进行比较分析。同比增长率 = (本期数 - 去年同期数) / 去年同期数 × 100% | NC财务系统 |
| 环比 | 与上期比,与上月比 | 与上一个期间进行比较分析。环比增长率 = (本期数 - 上期数) / 上期数 × 100% | NC财务系统 |
| 会计年度 | 财务年度 | 会计核算的年度区间。NC系统使用cyear字段存储，4位数字如2024 | NC财务系统 |

## 四、应收应付类术语（8个）

| 付款 | 付款单,供应商付款 | 企业向供应商支付款项的单据。主表：ap_paybill，有效付款单条件：billstatus >= 2（已审核）AND dr = 0 | NC财务系统 |
| 账龄 | 账龄分析,逾期天数 | 按照账款发生时间进行的分类统计。计算方式：当前日期 - 单据日期。通常分为：0-30天、31-60天、61-90天、91-180天、180天以上 | NC财务系统 |
| 客户 | 客户档案,往来客户 | 企业销售往来的客户信息。主表：bd_customer，主键：pk_customer，常用于应收账款的辅助核算 | NC财务系统 |
| 供应商 | 供应商档案,往来供应商 | 企业采购往来的供应商信息。主表：bd_supplier，主键：pk_supplier，常用于应付账款的辅助核算 | NC财务系统 |
| 预付账款 | 预付款,预付供应商款 | 企业预先支付给供应商的款项。科目编码：1123，正余额表示预付款待核销 | NC财务系统 |
| 预收账款 | 预收款,客户预付款 | 企业预先收取客户的款项。科目编码：2203，正余额表示预收款待核销 | NC财务系统 |
| 往来核销 | 核销,对账核销 | 将预付/预收款项与实际交易进行匹配抵消的过程。核销后余额清零 | NC财务系统 |

## 六、财务分析类术语（8个）

| 术语名称 | 同义词（逗号分隔） | 术语描述 | 数据源名称 |
|:---|:---|:---|:---|
| 资产负债率 | 负债比率,举债经营比率 | 衡量企业负债水平和风险程度的指标。计算公式：资产负债率 = 负债总额 / 资产总额 × 100%。资产总额：科目编码1xxx的期末余额合计，负债总额：科目编码2xxx的期末余额合计。参考值：一般不超过70% | NC财务系统 |
| 流动比率 | 流动资金比率 | 衡量企业短期偿债能力的指标。计算公式：流动比率 = 流动资产 / 流动负债。流动资产：科目编码1001-1299，流动负债：科目编码2001-2299。参考值：一般2:1较为合理 | NC财务系统 |
| 速动比率 | 酸性测试比率 | 更严格的短期偿债能力指标，排除存货。计算公式：速动比率 = (流动资产 - 存货) / 流动负债。存货科目：1401-1499。参考值：一般1:1较为合理 | NC财务系统 |
| 毛利率 | 销售毛利率,毛利润率 | 衡量产品盈利能力的指标。计算公式：毛利率 = (营业收入 - 营业成本) / 营业收入 × 100%。营业收入：科目5001贷方发生，营业成本：科目6001借方发生 | NC财务系统 |
| 净利率 | 销售净利率,净利润率 | 衡量企业最终盈利能力的指标。计算公式：净利率 = 净利润 / 营业收入 × 100%。净利润 = 收入类科目贷方 - 成本费用类科目借方 | NC财务系统 |
| 应收周转率 | 应收账款周转率 | 衡量应收账款回收速度的指标。计算公式：应收账款周转率 = 营业收入 / 平均应收账款。周转天数 = 365 / 周转率 | NC财务系统 |
| 收入 | 营业收入,销售收入,营收 | 收入类科目的发生额合计，通常指主营业务收入和其他业务收入。科目编码：5001（主营业务收入）、5051（其他业务收入）。计算：SUM(贷方发生额 - 借方发生额) | NC财务系统 |
| 利润 | 净利润,利润总额 | 收入减去成本费用后的金额。主要项目：营业收入 - 营业成本 - 期间费用。期间费用包括：6601销售费用、6602管理费用、6603财务费用 | NC财务系统 |

## 七、辅助核算类术语（4个）

| 部门费用 | 部门核算,成本中心费用 | 按部门维度进行的费用归集核算。关联表：org_dept，常用于管理费用、销售费用等科目 | NC财务系统 |

|:---|:---|:---|:---|
| ... | ... | ... | NC财务系统 |

> **提示**: SQLBot导入时会自动将术语向量化存入pgvector，用于语义检索匹配。
